# 제품 요구사항 정의서 (PRD)

## 1) 목적

- 해커톤 참가자가 **대회 탐색, 팀 구성, 제출, 결과 확인**을 한 곳에서 수행할 수 있는 웹 플랫폼 구축.
- 데이터 제공이 제한된 상황에서 **로컬 스토리지 + Supabase** 기반으로 빠르게 구현 가능한 구조 제공.

## 2) 핵심 사용자

- 참가자(개인/팀)
- 팀장(팀 생성/모집 관리)
- 운영자(향후 확장)
- 심사 이후 결과 정리/포트폴리오 제작이 필요한 사용자

## 3) 필수 라우트(MVP)

- `/` 메인
- `/hackathons` 해커톤 목록
- `/hackathons/:slug` 해커톤 상세
- `/rankings` 전체 랭킹
- `/camp` 팀 모집/팀 탐색

## 4) 해커톤 상세 필수 섹션

- Overview(개요)
- Info(안내/규칙)
- Eval(평가)
- Schedule(일정)
- Prize(상금)
- Teams(팀)
- Submit(제출)
- Leaderboard(리더보드)

## 5) 사용자 스토리

- 사용자는 메인에서 해커톤 보러가기/팀 찾기를 통해 빠르게 이동할 수 있다.
- 사용자는 해커톤 목록에서 카드 클릭 시 상세로 진입할 수 있다.
- 사용자는 상세의 Submit 섹션에서 항목을 저장/제출하고 리더보드 반영 상태를 확인할 수 있다.
- 사용자는 Camp에서 팀을 생성하고 모집 여부/연락 링크를 관리할 수 있다.
- 사용자는 Rankings 또는 상세 Leaderboard에서 순위를 확인할 수 있다.

## 6) 추가 기능(요청 반영)

1. 해커톤 일정/규칙 안내 AI 챗봇
   - 현재 선택된 해커톤의 `info.notice`, `schedule`, `submit.guide`, `eval` 정보 기반 응답.
2. 시각화 리더보드/대시보드
   - 점수 변화, 제출 시각, 순위 분포, 투표 점수 분해(참가자/심사위원) 표시.
   - 심사 이후 결과물/수료증 기록 저장.
3. GitHub README 기반 포트폴리오 HTML 생성
   - GitHub 계정 입력 -> README 파싱 -> UI 템플릿 적용 -> 단일 HTML 출력.
4. 랜딩페이지 3D 인터랙션
   - 첫 진입에서 3D 모션 기반 브랜딩 경험 제공.

## 7) 비기능 요구사항

- TypeScript strict 모드.
- 접근성(키보드 이동, 대비, aria-label) 준수.
- 에러 추적: Sentry.
- 테스트: Vitest + Playwright.
- 배포: Vercel.

## 8) 범위 관리

### MVP 우선순위

- 라우팅/기본 목록/상세/제출/리더보드/팀모집
- 로컬 스토리지 기반 CRUD

### Phase 2

- Supabase 연동(Auth + RLS + Postgres)
- AI 챗봇 고도화(RAG/컨텍스트 캐싱)
- README 포트폴리오 스타일 고도화
