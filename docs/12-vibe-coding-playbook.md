# 바이브 코딩 플레이북

## 1) 세션 시작 루틴

1. `docs/01-product-requirements.md` 확인
2. 오늘 구현 범위 1~2개로 제한
3. 완료 기준(DoD) 먼저 작성

## 2) 구현 우선순위

- UI 뼈대 -> 목데이터 연결 -> 사용자 액션 -> 검증
- 큰 기능은 세로(slice) 단위로 완성

## 3) 프롬프트 운영

- 요청은 "목표/제약/완료조건" 3줄로 작성
- 변경 요청은 이전 합의 문서 경로를 함께 명시
- 결과는 항상 `docs/prompt-log.md`에 누적

## 4) 리뷰 기준

- 요구사항 충족 여부
- 회귀 영향 범위
- 성능/접근성/오류 처리

## 5) Anti-pattern

- 문서 합의 없이 구조 대수술
- 테스트 없이 상태 관리 대규모 변경
- 하드코딩된 스키마/권한 규칙
