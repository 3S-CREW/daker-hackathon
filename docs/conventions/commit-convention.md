# 커밋/PR 컨벤션

## 1) Commit 메시지

Conventional Commits 사용

- `feat:` 기능 추가
- `fix:` 버그 수정
- `docs:` 문서 수정
- `refactor:` 리팩터링
- `test:` 테스트
- `chore:` 설정/빌드/의존성

예시:

- `feat(web): add hackathon detail tab routing`
- `fix(api): validate submission payload schema`

## 2) 브랜치 전략

- 브랜치 네이밍/운영 규칙은 `docs/conventions/branch-convention.md`를 따른다.

## 3) PR 규칙

- 작고 명확한 단위로 PR
- 체크리스트(테스트/문서/스크린샷) 작성
- Breaking change는 본문 최상단에 명시
