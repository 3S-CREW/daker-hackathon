1. 프로젝트 제목

- 해커톤 참가자와 팀장을 위한 커뮤니티형 운영 서비스, 해커톤 허브

2. 서비스 개요

- 서비스명: 해커톤 허브(Hackathon Hub)
- 한줄 소개: 해커톤 탐색, 팀 구성, 제출, 랭킹 확인, 성과 아카이빙까지 한 번에 처리하는 올인원 플랫폼
- 개발 목표
  - 명세서와 예시 데이터만으로도 빠르게 구현 가능한 실전형 해커톤 서비스 구축
  - 대회 참여 전(탐색/팀빌딩)부터 참여 후(결과 분석/포트폴리오화)까지 사용자 여정을 하나로 연결하는 서비스 제작
  - 로컬 저장소 기반 MVP를 우선 구현하고 Supabase 기반 운영형 구조로 확장
- 타겟 사용자
  - 해커톤 참가자(개인/팀)
  - 팀장(팀 생성/모집/제출 관리)
  - 심사 이후 성과 정리와 포트폴리오 제작이 필요한 사용자
- 사용자 가치(기대효과)
  - 대회 정보를 빠르게 이해하고 일정/규칙 실수를 줄일 수 있음
  - 팀 탐색/모집 동선을 단순화해 팀 결성을 빠르게 진행 가능
  - 제출물/결과/수료증을 누적 관리해 면접/포트폴리오 준비 시간을 단축

3. 페이지 구성

- 랜딩 페이지 (/)
  - 3D 인터랙션 히어로 영역과 핵심 CTA(해커톤 보러가기, 팀 찾기, 랭킹 보기, 포트폴리오 제작)를 상단 네비게이션바에 제공
  - 사용자 행동: 버튼 클릭으로 /hackathons, /camp, /rankings 등으로 이동
- 해커톤 목록 (/hackathons)
  - 해커톤 카드(제목, 상태, 기간 등)를 표시
  - 사용자 행동: 카드 클릭 시 /hackathons/:slug 상세 진입, 상태/태그 필터 적용
- 해커톤 상세 (/hackathons/:slug)
  - 섹션: Overview, Info, Eval, Schedule, Prize, Teams, Submit, Leaderboard
  - 사용자 행동: 제출 저장/제출, 팀 모집 페이지 이동, 규정/FAQ 링크 확인, 챗봇 질의응답
- 랭킹 페이지 (/rankings)
  - 대회별/글로벌 랭킹 테이블과 점수 기반 정렬 표시
  - 사용자 행동: 해커톤 선택 필터, 특정 팀 상세 확인
- 팀 모집 페이지 (/camp)
  - 팀 리스트, 모집 상태, 포지션, 소개, 연락 링크를 표시
  - 사용자 행동: 팀 생성, 모집 상태 변경, 대회별 필터링, 연락 링크 이동
- 개인 대시보드(확장: /dashboard)
  - 참가 이력, 제출물, 점수 추이, 회고 메모, 수료증 보관함 제공
  - 사용자 행동: 기록 등록/수정, 수료증 업로드 또는 URL 저장
- 포트폴리오 빌더(확장: /portfolio)
  - GitHub 계정/저장소 입력 후 README 기반 포트폴리오 생성
  - 사용자 행동: 테마 선택, HTML 내보내기

페이지 연결관계 요약

- / -> /hackathons, /camp, /rankings
- /hackathons -> /hackathons/:slug
- /hackathons/:slug (Teams 섹션) -> /camp?hackathon=:slug
- /hackathons/:slug (Submit/Leaderboard) -> 제출 결과 및 랭킹 반영
- /rankings, /dashboard, /portfolio 는 상세/제출 결과 데이터와 연동

4. 시스템 구성

- 프론트엔드: React + TypeScript + Vite + React Router + Tailwind + shadcn/ui + Framer Motion
- 상태관리
  - 서버 상태: TanStack Query
  - 클라이언트 상태: Zustand
  - 폼: React Hook Form + Zod
- 데이터 계층
  - 1차: localStorage 기반 CRUD(MVP)
  - 2차: Supabase PostgreSQL + PostgREST + Auth + RLS
- 백엔드 확장
  - Supabase Edge Functions(Deno)로 AI 챗봇 프록시, README 파싱/정규화 처리
- 운영/품질
  - Vercel 배포, GitHub Actions CI, Sentry 모니터링, Vitest/Playwright 테스트

5. 핵심 기능 명세

- 기능 1: 해커톤 목록/상세 조회
  - 입력: 해커톤 카드 클릭 또는 URL slug
  - 동작: slug 기반으로 상세 데이터 로드 및 8개 섹션 렌더링
  - 결과: 사용자는 규칙/일정/평가/제출 방법을 한 화면에서 확인
- 기능 2: 팀 모집/팀 생성(camp)
  - 입력: 팀명, 소개, 모집여부, 필요 포지션, 연락 링크
  - 동작: 유효성 검사 후 저장(localStorage 또는 Supabase)
  - 결과: 팀 목록 즉시 갱신, 해커톤별 필터로 탐색 가능
- 기능 3: 제출(Submit) 및 리더보드 반영
  - 입력: 대회 제출 항목(기획서/웹링크/PDF 등)
  - 동작: 항목 저장 후 제출 확정 시 리더보드 데이터 업데이트
  - 결과: 팀 점수/순위가 Leaderboard 및 Rankings에 반영
- 기능 4: 해커톤 안내 AI 챗봇
  - 입력: 선택 해커톤 slug + 사용자 질문
  - 동작: 일정/규칙/제출/평가 데이터를 컨텍스트로 OpenAI API 질의
  - 결과: 근거 기반 답변 제공, 모호한 질문은 rules/faq 링크 안내
- 기능 5: 리더보드/대시보드 시각화
  - 입력: 제출/점수/투표 분해 데이터
  - 동작: 순위 막대, 제출 타임라인, 점수 구성 차트 생성
  - 결과: 참가자는 성과를 한눈에 파악하고 회고 데이터 누적 가능
- 기능 6: 수료증/성과 보관
  - 입력: 수료증 파일 또는 URL, 메타데이터(대회명/날짜/설명)
  - 동작: 사용자 계정 단위 저장 및 목록화
  - 결과: 면접/포트폴리오용 자료를 즉시 열람/공유 가능
- 기능 7: GitHub README 기반 포트폴리오 생성
  - 입력: GitHub 계정/저장소, 테마 선택
  - 동작: README 파싱 -> 섹션 매핑 -> HTML 템플릿 렌더
  - 결과: 다운로드 가능한 포트폴리오 HTML 생성

6. 주요 사용 흐름

- 흐름 1: 해커톤 탐색 -> 상세 확인 -> 팀 합류
  1. 사용자는 랜딩에서 해커톤 보러가기를 클릭
  2. 목록에서 해커톤 카드를 선택해 상세로 진입
  3. Teams 섹션에서 camp로 이동해 팀 탐색/연락
  - 예외 처리:
    - 데이터 없음: 빈 상태 카드 + 새로고침/초기 데이터 로드 안내
    - 네트워크/접속 불가: 오류 메시지 + 재시도 버튼 표시
- 흐름 2: 제출 진행 -> 리더보드 확인
  1. 상세 Submit 섹션에서 제출 항목 입력
  2. 저장 후 제출 버튼으로 최종 반영
  3. Leaderboard 섹션에서 점수/순위 확인
  - 예외 처리:
    - 입력 오류: 필수값 누락/형식 오류를 필드 단위로 표시
    - 제출 제한 초과: 대회 규정 안내와 함께 제출 차단 메시지 표시
- 흐름 3: AI 챗봇으로 규칙/일정 확인
  1. 사용자가 챗봇에 질문 입력
  2. 챗봇이 현재 해커톤 컨텍스트로 답변 생성
  3. 답변과 함께 근거 섹션/링크 제공
  - 예외 처리:
    - 답변 불가: 규정 링크 및 운영 문의 경로 안내
- 흐름 4: 성과 정리 -> 포트폴리오 생성
  1. 대시보드에서 제출/점수/수료증 기록 정리
  2. 포트폴리오 빌더에서 README 기반 HTML 생성
  3. 결과물을 저장하고 외부 공유
  - 예외 처리:
    - README 파싱 실패: 입력 저장소 확인 요청 + 수동 입력 대체 UI 제공

7. 개발 및 개선 계획

- Phase 1 (MVP, 1주)
  - 범위: 랜딩, 목록, 상세 8섹션, camp, rankings, submit 기본, localStorage
  - 목표: 핵심 사용자 여정(탐색-팀구성-제출-확인) 동작 완료
- Phase 2 (운영형 전환, 1주)
  - 범위: Supabase 스키마/RLS/Auth, PostgREST 연동, CI 파이프라인 강화
  - 목표: 계정 기반 데이터 격리와 안정적 배포 체계 구축
- Phase 3 (AI/성과관리 고도화, 1주)
  - 범위: 해커톤 챗봇, 대시보드 시각화, 수료증 보관, README 포트폴리오 생성
  - 목표: 참여 이후 성과 활용 가치를 높이는 차별화 기능 제공

우선순위:

- P0: 라우팅/상세/제출/랭킹/camp
- P1: Auth/RLS, 안정화 테스트, 운영 자동화
- P2: AI 챗봇, 포트폴리오 생성, 고급 분석 UX
